```
 自分のブログサイトをフロントエンドとバックエンド分離するによって、サイトのインタフェースを高めたことである。
  従来の一体型サイトはブラウザから一部のデータをリクエストする時、サーバーがサイト全体をもう一度レンダリングになることによって、再読み込み必要がある。ユーザーインタフェースの体験が悪くなるので、ブログのアクセス数が平均毎月１０％下がり続けてる。
  私は自分のブログをアクセスする友達の意見をヒアリングした。それらの意見の中で私はサイトの読み込みと応答時間が長いということが課題と感じ
  ２つの対策を取り上げた。１つ目はコードの高凝集と低結合であり、２つ目はデカップリングしたコードとフロントエンドのモジュール化の連携であった。
  その連携をすることで,ユーザーインタフェース改善できるようになっており、結果として前と比べサイトの読み込み時間が７０％減少し、アクセス数が２０％上がった。
```

## 学校を選んだ理由

+ 高校・大学 学校を選んだ理由とその判断基準

文理融合です。文系理系を選んだ時、自分本当に迷ってた。文系と理系両方が好きです

成績も全部同じです。(数学と国語も同じ程度好きです)。両方の視野からこの世界を見に行きたい。文系の柔軟性と理系の論理性を両方勉強したいので、今の経営システム工学を選んだ。

合格したmarchの理系の中で　中央大学を選んだ。同じmarch理系ので、教授、設備、研究なども差がないが、でも中央大学は私の初めて受けた大学だ。それに初めて合格した大学です。1~10は難しくないかもしれないが、0から1は本当に辛いです。最初にチャンスをくれた母校に本当に感謝します。恩を返すみたいな感じで、大学入って、自分なりに頑張ります。
